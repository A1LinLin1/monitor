Monitor Recorder - 全能型 Windows 活动记录器

Monitor Recorder 是一款基于 C++ 和 Qt 6 框架开发的轻量级 Windows 系统监控工具。它结合了底层 Windows Hook（钩子）技术与现代化的图形用户界面，能够精准地捕捉键盘输入、识别当前活跃窗口，并提供人性化的日志检索功能。

🌟 核心特性

全系统监控：利用底层全局钩子技术，能够跨进程捕获系统范围内的按键和鼠标活动。

智能字符转换：自动将物理按键映射为可读的 ASCII 字符，直观展示输入内容（如：按下 Shift+A 显示为 A）。

多维度筛选检索：

窗口筛选：无需手动输入进程名，系统自动提取已记录的窗口列表供用户勾选过滤。

时间回溯：支持按特定时间段精确查询历史操作日志。

高性能架构：

事件分离：监控逻辑通过 DLL 注入实现，主界面负责展示，保证了系统的稳定性和响应速度。

低资源占用：原生 C++ 编写，内存占用极低。

高度可定制化：支持通过外部 style.css (QSS) 文件深度定制 UI 界面（颜色、字体、布局等）。

🏗️ 项目架构

项目采用模块化设计，主要分为以下两个部分：

监控核心 (myhook)：

编译生成 myhook.dll。

负责调用 SetWindowsHookEx 接口，通过共享内存（Shared Section）与主程序通信。

管理终端 (recorder_ui)：

基于 Qt 6 开发的 GUI 程序。

负责数据的实时渲染、日志持久化存储以及用户交互。

🛠️ 构建与安装

环境要求

操作系统：Windows 10 / 11 (x64)

编译器：MSVC 2019 / 2022

框架：Qt 6.5.0 或更高版本

构建工具：CMake 3.20+

编译步骤

克隆仓库：

git clone [https://github.com/A1LinLin1/monitor.git](https://github.com/A1LinLin1/monitor.git)
cd monitor


使用 CMake 构建：

mkdir build && cd build
# 请将路径替换为您本地的 Qt 安装路径
cmake .. -DCMAKE_PREFIX_PATH="C:/Qt/6.5.0/msvc2019_64"
cmake --build . --config Release


运行：确保生成的 myhook.dll 与 recorder_ui.exe 处于同一目录下。

📖 使用说明

启动监控：运行程序后，点击“开始”按钮。建议以管理员身份运行，以确保能够获取到高权限窗口（如任务管理器）的输入信息。

日志过滤：在界面右侧或底部的筛选栏中，可以从下拉列表中勾选特定的“窗口名称”，实时流将仅显示该窗口下的活动。

界面定制：如果你想更换主题，可以直接编辑根目录下的 style.css。保存后重启程序，新的样式即可生效。

日志导出：所有记录会自动保存在程序运行目录下的日志文件中。
